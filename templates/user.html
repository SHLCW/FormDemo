This tutorial is meant to teach you about some basic web concepts such as HTML5, forms, requests, responses, POST / GET, and cookies.

Additionally, this also covers some less generic concepts and specific libraries such as Flask, Bootstrap, Jinja2 templates, some general Python work, and GIT. GIT, of course, is the source control mechanism used for working our way through the lesson. We check out each section as marked by tags. To save our work, we add items to our GIT workspace, commit changes, and then push to our 'origin' repository (in our case, this is Github.com). PyCharm makes this GIT stuff easy by showing filenames that haven't been added to the local GIT repo in red. If the '.gitignore' file marks a folder or file to be excluded, PyCharm shows it as gray. Newly added files are shown as green, and modified files are shown as blue. When you sit down to work, don't forget to pull from your origin. You might have modified the code last on a different machine, or another authorized developer might have pushed a change.

Web Resources

  • W3Schools One of the best reference and learning sites for general web technologies such as HTML, CSS, JavaScript, and even some specific libraries like Bootstrap and JQuery.
  • Heroku
  • Learn JavaScript Sub Reddit Reddit is full of interesting topics. Pick one you like and go! The learn JavaScript link is the place where you can post questions even if you don't have time to put as much structure into it for something like a question on StackOverflow.
  • StackOverflow If you Google a question and one of the results is a StackEchange site, then that's likely the best answer. Most of the time, StackOverflow is the one you'll be interested in since it covers question and answer for specific programming problems, software algorithms, coding techniques, software development tools, but there is also a StackExchange for Code Review, Super User (questions about enterprise desktop software), Server Fault (for system administrators and IT professionals, people who manage or maintain computers in a professional capacity), and many more. The only downside to these sites is that if you ask your own question, you need to follow the rules for question quality. Take the tours on any of these sites when you create your account, and you'll be all set.

Python and Flask

  • Python.org Docs The official documentation is quite good. While sometimes a Google search followed by clicking the first Stack Overflow link will get you more obscure specific advice, if you're wondering about usage of the standard library, you should consider looking here first. See the module docs for SQLite3 or built-in types.
  • Flask Docs This guide is easy to follow, possibly in part due to the simplicity of Flask itself. The Quickstart Guide is a pretty good place to learn.
  • Flask Mega Tutorial Miguel Grinberg has a fantasic tutorial on Flask which has now been published as a book as well. Like this demo project, it is broken into sections using Git tags.
  • Advanced SQLite Python Central has a lot of good tutorials on advanced topics such as SQLAlchemy (an ORM not used in this tutorial), but this particular tutorial talks about a number of great subjects such as defining a SQL function in Python, dealing with dates in SQL as objects once they are in Python, and the executescript method.
